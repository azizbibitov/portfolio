Overview

What is Cl:k?

Cl:k is a social platform built around intentional sharing — letting users choose exactly who sees their content through VIP and Circle story tiers.

📺 VIP & Circle Stories

Two distinct story tiers let users share content with different audiences. VIP stories are for close connections, Circle stories for a wider network. Each tier has its own visual identity and interaction model.

💬 Rich Messaging

Full-featured chat with voice notes, GIF support via GIPHY, link preview cards powered by Apple's LinkPresentation framework, and real-time unread badges.

🔔 Smart Notifications

Real-time notification bell updates via socket events. Story-related notifications automatically expire after 24 hours. Push notification deep-linking opens the correct space theme.

🔗 Share Extension

iOS Share Extension lets users share content from any app directly into Cl:k. Implemented silent token refresh so the extension works even when the access token has expired — no need to open the main app first.

My Contributions

Work Done

I worked on a wide range of tasks spanning UI, features, bug fixes, and performance improvements across the full codebase.

Task 01

Circle Space UI Card Redesign

Redesigned Circle story cells to be fully circular with a double border ring — teal gradient for unwatched, grey when all-watched, no border when the friend has no story. Implemented screen-height-based sizing so the layout scales across all device sizes.

Task 02

Message URL Link Preview

Built link preview cards inside message bubbles using Apple's LinkPresentation framework. Implemented a two-layer cache (NSCache + disk) so previews persist across cold launches and cell reuse never shows stale content.

Task 03

GIF Support in Comments

Integrated the GIPHY SDK for comment GIFs. GIF cells use dynamic height from each image's aspect ratio and a static height cache so cells never jump when scrolled back into view. Handled reply mode, GIF button visibility, and multipart upload.

Task 04

Share Extension Token Refresh

Fixed 401 errors in the Share Extension by implementing silent token refresh using shared App Group storage, with automatic retry of the failed request — no need to open the main app after token expiry.

Task 05

Audio Recording in Chat

Built MVP voice note recording in the conversation screen with AVAudioSession management. Updated the voice note UI to visually differentiate sent and received messages. Fixed an iOS 15 audio session error with a proper activation fallback.

Task 06

Conversation Header Redesign

Replaced the plain nav bar title with a card-style header showing the friend's profile picture and name. Added a fullscreen viewer that opens when tapping a friend's display picture, and fixed profile navigation from the header.

Task 07

Story Features & Bug Fixes

Loading timeout and retry for the story player, correct space theme from push notification deep links, story progress bar via notification, caption overlap fix, silent mode audio handling, and overlapping toast message fix on story reply.

Task 08

Notification & Badge System

Real-time notification bell via socket events, unread count badge on the Messages tab, story thumbnail on the notification screen, and automatic expiry of story notifications older than 24 hours.

Task 09

Comment Enhancements

Comment count indicators on story previews, mention shape fix on the comment screen, GIF icon size fix, multi-line caption text field with enhanced styling, and textBio cursor overflow fix on multiline input.

Task 10

Onboarding — Add & Invite Friends

Built a new onboarding screen after profile setup prompting users to add or invite friends. Add to VIP and Add to Circle cards continue showing until the user reaches their friend limit, improving new-user discoverability.

Task 11

Media Download to Gallery

Changed media download behaviour so files are saved directly to the iPhone Photos gallery instead of the Files app, matching standard social app expectations.

Task 12

Crash Fixes & Stability

Fixed an app crash when tapping a comment thumbnail (EXC_BAD_ACCESS from a mismatched XIB custom class), suppressed a spurious invalid access token error on the front end, and resolved multiple AutoLayout-related crashes triggered during live table layout passes.
